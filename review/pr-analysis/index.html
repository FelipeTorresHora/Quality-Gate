Por que a análise falha em produção (resposta curta)

O pipeline de análise (execute_analysis_run) executa clone + instalação de dependências + suíte de testes + scanners + IA de forma síncrona dentro da requisição HTTP, e depende de binários externos (npm, node, go, git, pytest, scanners) rodando via subprocess. O deploy de produção é serverless na Vercel (vercel.json → experimentalServices.backend), onde:

  • O Dockerfile (que instala git/go/node/npm) não é usado — a Vercel usa o builder Python dela, sem esses binários.
  • O sistema de arquivos é somente-leitura exceto /tmp, então o install_command padrão (pip install -r requirements.txt) falha.
  • maxDuration = 300s, mas command_timeout_seconds = 600s e o pipeline roda inteiro na request → a função é morta antes de terminar → 504 / FUNCTION_INVOCATION_TIMEOUT.
